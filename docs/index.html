Architecture
How it works
01
01
Messages arrive

Producers push messages to an AWS SQS queue. A Dead Letter Queue (DLQ) captures messages that fail after 5 retries.

02
02
KEDA polls the queue

KEDA's ScaledObject checks the queue depth every 30s. It calculates the desired replica count as ⌈messages / queueLength⌉.

03
03
EKS scales the Deployment

KEDA adjusts the HPA target. EKS spawns or terminates pods instantly — scaling to zero when the queue drains to save cost.

04
04
IRSA authentication

The consumer pod uses a Kubernetes ServiceAccount linked to an IAM role via IRSA — no hardcoded credentials, ever.

Deploy
Quick start
bash — setup & deploy
# 1. Provision AWS resources (idempotent Python script)
$ export CLUSTER_NAME="my-eks-cluster"
$ python3 aws_resources/setup_aws_resources.py
✓ DLQ created: https://sqs.us-east-1.amazonaws.com/.../keda-with-sqs-dlq
✓ Main queue created: https://sqs.us-east-1.amazonaws.com/.../keda-with-sqs
✓ IAM policy created: arn:aws:iam::...policy/keda-with-sqs-policy
✓ Service account created/updated

# 2. Install KEDA on EKS
$ helm repo add kedacore https://kedacore.github.io/charts
$ helm install keda kedacore/keda --namespace keda --create-namespace

# 3. Deploy the consumer and ScaledObject
$ kubectl apply -f app-k8s/app-k8s.yaml
$ kubectl apply -f app-k8s/scale.yaml

# 4. Watch pods scale in real time
$ kubectl get pods -w
